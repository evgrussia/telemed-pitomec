# Матрица стейкхолдеров

Анализ ключевых сторон, влияющих на успех проекта Telemed Pitomec.

## 1. Карта стейкхолдеров (Power/Interest Grid)

```
                    ИНТЕРЕС (INTEREST)
              Низкий           Высокий
         ┌──────────────┬──────────────┐
    Выс. │   KEEP       │   MANAGE     │
         │   SATISFIED  │   CLOSELY    │
ВЛИЯНИЕ  │ (Удовлетворять)│ (Плотно раб.)│
 (POWER) ├──────────────┼──────────────┤
    Низ. │   MONITOR    │   KEEP       │
         │ (Мониторить) │   INFORMED   │
         │              │ (Информировать)│
         └──────────────┴──────────────┘
```

| Стейкхолдер | Влияние | Интерес | Категория |
|-------------|---------|---------|-----------|
| Фаундеры / Инвесторы | High | High | Manage Closely |
| Ветеринары-партнеры | High | High | Manage Closely |
| Владельцы животных (Юзеры) | Low | High | Keep Informed |
| Команда разработки | Medium | High | Manage Closely |
| Регуляторы (РКН, Россельхознадзор) | High | Low | Keep Satisfied |
| Конкуренты | Medium | Medium | Monitor |
| Платформа Telegram | High | Low | Monitor / Keep Satisfied |

## 2. Реестр и стратегия взаимодействия

| ID | Стейкхолдер | Интересы | Риски / Ожидания | Стратегия |
|----|-------------|----------|------------------|-----------|
| **SH-01** | **Фаундеры** | ROI, рост MAU, лидерство в нише AI-вет. | Ожидают соблюдения сроков и качества AI. | Еженедельные отчеты по метрикам и roadmap. |
| **SH-02** | **Врачи** | Доп. доход, удобный инструмент работы, репутация. | Боятся юридической ответственности за онлайн-советы. | Понятные договоры, страхование ответственности, обучение работе с AI. |
| **SH-03** | **Владельцы животных** | Здоровье питомца, экономия времени/денег, удобство 24/7. | Хотят точности от AI и быстрой связи с врачом. | Интуитивный UX в Telegram, прозрачность работы AI (дисклеймеры). |
| **SH-04** | **Регуляторы** | Соблюдение законов о данных и ветеринарии. | Могут ужесточить правила телемедицины. | Legal-compliance, хранение данных в РФ, четкое разграничение "совет" vs "диагноз". |
| **SH-05** | **Команда разработки** | Технологический стек, интересные задачи (AI). | Технический долг, сложность интеграции AI. | Регулярные спринты, ADR для ключевых решений, адекватный бэклог. |

| Стейкхолдер | Канал | Частота | Ответственный |
|-------------|-------|---------|---------------|
| Фаундеры | Zoom / Документы | Weekly | PM / Analyst |

| Юзеры | Telegram Channel / Push | Regular updates | Marketing / Content |
| Команда | Jira / Slack (TG) | Daily Standups | Tech Lead |

---

## 4. Ключевые решения по взаимодействию

1. **Для врачей**: Разработать "Ветеринарный кабинет" внутри WebApp, который будет максимально упрощать заполнение протокола осмотра с помощью AI-ассистента.
2. **Для пользователей**: Внедрить систему рейтингов и отзывов для формирования доверия к врачам-партнерам.
3. **Для регуляторов**: Оформить использование AI как "информационную поддержку" (triage), а не медицинскую услугу, до уточнения законодательства.
